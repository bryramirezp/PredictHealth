📚 Documentación Técnica

Bienvenido a la documentación técnica de PredictHealth. Aquí encontrarás información detallada sobre la arquitectura, tecnologías, y componentes del sistema de analítica predictiva de salud.

¿Qué es PredictHealth?

PredictHealth es una plataforma de IA para predicción de enfermedades crónicas (Equipo 3). Plataforma que predice riesgos de enfermedades como diabetes e hipertensión usando IA y datos personales de salud. El objetivo es implementar medidas preventivas personalizadas para reducir enfermedades crónicas.

Datos Utilizados

  • Historial médico
  • Estilo de vida
  • Datos genéticos
  • GPS

Tecnologías

  • Backend: Flask, microservicios en FastAPI
  • Bases de datos: Firebase + PostgreSQL + Redis
  • App móvil: Android Java
  • Procesamiento de Datos: Modelos ML en Hadoop para clasificar riesgo
  • Visualización: Paneles personalizados y simulación con LeapMotion

Aplicación Real

  • Clínicas preventivas
  • Aseguradoras
  • Salud pública

Características Principales

🔐 Autenticación y Autorización

Sistema robusto de autenticación basado en JWT (JSON Web Tokens) con roles diferenciados para pacientes, doctores e instituciones médicas.

🤖 Modelos Predictivos

Modelos de machine learning entrenados con datos clínicos reales para predecir:

  • Riesgo de enfermedades cardiovasculares
  • Probabilidad de desarrollar diabetes tipo 2
  • Factores de riesgo personalizados

📊 Dashboard Interactivo

Visualización de datos en tiempo real con gráficas interactivas, métricas clave y recomendaciones personalizadas.

🔄 Arquitectura de Microservicios

Sistema modular y escalable diseñado para manejar múltiples servicios independientes:

  • Servicio de autenticación
  • Servicio de predicciones ML
  • Servicio de gestión de pacientes
  • Servicio de análisis de datos

Arquitectura del Sistema

El sistema sigue una arquitectura de tres capas con separación clara de responsabilidades:

  1. Capa de Presentación: Interfaces web y móvil para usuarios finales
  2. Capa de Lógica de Negocio: API REST con Flask y microservicios
  3. Capa de Datos: PostgreSQL para persistencia y Redis para caché

Primeros Pasos

Para comenzar a trabajar con PredictHealth, te recomendamos seguir estos pasos:

  1. Revisar la arquitectura del sistema
  2. Configurar el backend Flask
  3. Entender los modelos de ML
  4. Explorar el frontend
