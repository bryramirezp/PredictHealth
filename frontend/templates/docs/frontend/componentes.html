🧩 Componentes Frontend

Los componentes son las piezas básicas que construyen la interfaz de usuario de PredictHealth. Cada componente está diseñado para ser reutilizable y consistente en toda la aplicación.

¿Qué son los componentes?

Imagina que construir una aplicación web es como armar un rompecabezas. Los componentes son las piezas individuales que encajan perfectamente para crear la imagen completa. En lugar de dibujar cada pieza desde cero, usamos componentes prefabricados.

¿Por qué usar componentes?

  • Reutilización: Un componente se puede usar en muchas páginas
  • Consistencia: Todos los botones se ven igual
  • Mantenimiento: Cambiar un componente actualiza toda la aplicación
  • Velocidad: Los desarrolladores trabajan más rápido

Tipos de componentes

Componentes de navegación

Estos componentes ayudan a los usuarios a moverse por la aplicación. Incluyen menús, barras de navegación y breadcrumbs.

Componentes de contenido

Muestran información al usuario. Incluyen tarjetas, tablas, listas y paneles de información.

Componentes de formulario

Permiten a los usuarios ingresar información. Incluyen campos de texto, selectores, checkboxes y botones de envío.

Componentes de feedback

Informan al usuario sobre el estado de la aplicación. Incluyen mensajes de éxito, error, carga y confirmaciones.

Componentes principales de PredictHealth

Main Menu

Barra lateral de navegación con enlaces a todas las secciones.

Stat Cards

Tarjetas que muestran estadísticas importantes con iconos y números.

Data Tables

Tablas organizadas para mostrar listas de datos como pacientes o citas.

Form Components

Campos de entrada, selectores y validaciones para formularios.

Modals

Ventanas emergentes para confirmaciones y información adicional.

Action Buttons

Botones para realizar acciones como guardar, editar o eliminar.

Cómo crear un componente

Crear un componente es simple. Solo necesitas escribir el HTML con las clases CSS apropiadas. Por ejemplo, para crear una tarjeta de estadísticas: 

<div class="stat-card">
    <div class="stat-icon appointments">
        <i class="fas fa-calendar-check"></i>
    </div>
    <div class="stat-number" id="appointments-count">2</div>
    <div class="stat-label">Citas Programadas</div>
</div>

Convenciones de nomenclatura

Todos los componentes siguen reglas de nomenclatura consistentes para facilitar el mantenimiento:

Clases CSS

  • kebab-case: Para nombres de clases (stat-card, main-menu)
  • BEM: Block Element Modifier (stat-card__icon--active)
  • Prefijos: Para evitar conflictos (btn-primary, text-muted)

IDs de elementos

  • camelCase: Para identificadores únicos (patientName, appointmentsCount)
  • Descriptivos: Deben explicar claramente su propósito

Buenas prácticas

Para mantener la calidad del código y la consistencia de la interfaz, seguimos estas prácticas:

Diseño

  • Usar siempre las variables CSS definidas
  • Mantener proporciones y espaciado consistente
  • Asegurar accesibilidad (contraste, tamaños de toque)
  • Probar en diferentes dispositivos y navegadores

Código

  • Separar HTML, CSS y JavaScript
  • Usar clases semánticas y descriptivas
  • Evitar estilos inline
  • Comentar código complejo

Los componentes están diseñados siguiendo principios de usabilidad y accesibilidad para ofrecer la mejor experiencia posible.
